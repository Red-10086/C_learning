## 一、采集策略：四不原则

1. **不从头啃书**：不要打开《明解C中级篇》逐页精读，那不是你现在需要的。
2. **不写完整项目**：每个果实只写最小可行示例（10-30行），够验证直觉即可。
3. **不深究边界**：暂时不必测绘 `malloc` 失败的处理、链表删除的所有情况。知道“这里有边界”就够了。
4. **不强求一次精通**：采集目标是建立印象，不是立即应用。真正的精进发生在后续数据结构实战中。

## 四、注意事项

1. **课程资源**：优先使用翁恺的“散佚”章节（B站可搜），这些是原课程中为降低难度而移除的部分，对你而言恰恰是“刚刚进阶”的完美台阶。
2. **中级篇不是墓地**：不要害怕《明解C中级篇》——你只需要读它的第5、9、10章（动态内存、文件处理、综合项目）。它和你采集的锚点完全重叠。
3. **边界意识**：你已经在C基础中培养起了测绘边界的习惯。在采集阶段，可以暂时不测绘这些新概念的边界（比如 `malloc` 失败的处理）。等你进入数据结构实战时，再回头用公理化压路机轧平它们。
4. **知识库联动**：每摘完一个果实，用你已有的知识库模板记录一句话描述 + 一句口头表达。这会在你未来面试中被问到“用过动态内存吗？”时，让你能自然地讲出你当初的直觉和后来的验证。
